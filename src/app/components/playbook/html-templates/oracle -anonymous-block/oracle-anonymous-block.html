Migrating from: Oracle Anonymous Block

Overview

Oracle’s PL/SQL is the procedural extension of SQL. The PL/SQL program structure divides the

code into blocks which can be distinguished by the following key words:

DECLARE, BEGIN, EXCEPTION and END.

A PL/SQL code block that was not named (=not stored in the database as a procedure, function or package) is known as an anonymous block. The anonymous block serves as the basic unit of Oracle PL/SQL which preserves the following code sections:

• The Declarative Section (Optional)

Allows the user to declare variables by giving them names, data types and initial values.

• The Executable Section (Mandatory)

Each block structure must have at least one executable statement (SQL & PL/SQL).

• The Section (Optional)

Allows the user to add elements to handle exceptions or errors in the code.

Examples

Oracle Anonymous Block basic structure:

SQL> SET SERVEROUTPUT ON;

SQL> BEGIN

DBMS_OUTPUT.PUT_LINE('hello world');

END;

/

hello world

PL/SQL procedure successfully completed.

Oracle PL/SQL Anonymous block can have more advanced code elements such as the use of functions, cursors, dynamic SQL, conditional logics etc. The following anonymous block example holds the use of a cursor, conditional logics and exception-handling:

SQL>

SET SERVEROUTPUT

ON;

SQL>

DECLARE

NUMBER;

 

v_sal_chk

 

v_emp_work_years

NUMBER;

 

v_sql_cmd

VARCHAR2(2000);

BEGIN

FOR v IN (SELECT EMPLOYEE_ID, FIRST_NAME||' '||LAST_NAME AS EMP_NAME, HIRE_DATE, SALARY FROM EMPLOYEES)

LOOP

v_emp_work_years:=EXTRACT(YEAR FROM SYSDATE) - EXTRACT (YEAR FROM v.hire_date);

IF v_emp_work_years>=10 and v.salary <= 6000 then DBMS_OUTPUT.PUT_LINE('Consider a Bonus for: '||v.emp_name);

END IF;

END LOOP;

EXCEPTION WHEN OTHERS THEN DBMS_OUTPUT.PUT_LINE('CODE ERR: '||sqlerrm);

END;

/

The above code checks each employee for their work period by years, according to the

HIRE_DATE column (check if the values are greater or equal to 10 years) with a salary that is

lower or equal to 6000. If it answers both conditions, then the employee is a candidate for a bonus (this will be present in the code output using the dbms_output.put_line function).

For additional information on Oracle Anonymous Block:

https://docs.oracle.com/cd/B28359_01/appdev.111/b28370/controlstructures.htm#CJAEDEIH

Migration to : PostgreSQL DO

Overview

PostgreSQL version 9.6 supports capabilities similar to Oracle’s anonymous blocks. In PostgreSQL, you can run a PL/pgSQL code that is not stored in the database as an independent code segment using PL/pgSQL “ DO”.

PostgreSQL DO

PostgreSQL PL/pgSQL with can execute an anonymous code block.

PL/pgSQL is PostgreSQL extension to the ANSI SQL which has many similar elements to Oracle PL/SQL. PostgreSQL DO also uses the same code structure as Oracle anonymous block:

• The Declarative Section (Optional)

• The Executable Section (Mandatory)

• The Section (Optional)

Examples

1. PostgreSQL DO simple structure:

psql=> SET CLIENT_MIN_MESSAGES = 'debug'; Equivalent To Oracle SET SERVEROUTPUT ON

psql=> DO $$ BEGIN

RAISE DEBUG USING MESSAGE := 'hello world'; END $$;

DEBUG: hello world

DO

PostgreSQL PL/pgSQL DO supports the use of advanced code elements such as functions, cursors, dynamic SQL, conditional logics etc.

2. Shown here is a more complex PL/pgSQL do code structure, converted from Oracle’s “employee bonus”PL/SQL anonymous block example presented earlier in this topic:

psql=> DO $$ DECLARE

v_sal_chk DOUBLE PRECISION; v_emp_work_years DOUBLE PRECISION; v_sql_cmd CHARACTER VARYING(2000); v RECORD;

BEGIN FOR v IN

SELECT employee_id, CONCAT_WS('', first_name, ' ', last_name) AS emp_name, hire_date, salary

FROM employees LOOP

v_emp_work_years := EXTRACT (YEAR FROM now()) - EXTRACT (YEAR FROM v.hire_date);

IF v_emp_work_years >= 10 AND v.salary <= 6000 THEN

RAISE DEBUG USING MESSAGE := CONCAT_WS('', 'Consider a Salary Raise for: ', v.emp_name);

END IF;

END LOOP; EXCEPTION

WHEN others THEN

RAISE DEBUG USING MESSAGE := CONCAT_WS('', 'CODE ERR: ', SQLERRM);

END $$;

For additional information on PostgreSQL DO:

https://www.postgresql.org/docs/current/static/sql-do.html
